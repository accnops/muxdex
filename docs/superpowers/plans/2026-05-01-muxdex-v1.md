# Muxdex V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single-binary Rust TUI that discovers tmux sessions, renders one tile per session, preserves dead-session snapshots, supports resurrection by name, and allows confirmed session kill or dead-tile dismissal.

**Architecture:** Use a small, testable Rust library behind a thin `main.rs`. Split responsibilities into a `tmux` probe, `store` reconciliation layer, `layout` engine, and `ui` application loop so each behavior from the spec has a direct home and unit coverage. Treat tmux as an external data source behind a command runner abstraction so tests can exercise parsing and reconciliation without requiring a live tmux server.

**Tech Stack:** Rust, `ratatui`, `crossterm`, `anyhow`, `serde`, `unicode-width`

```rust
pub fn reconcile(&mut self, live: Vec<LiveSession>, now: Instant) {
    let live_names: HashSet<String> = live.iter().map(|item| item.name.clone()).collect();
    for item in live {
        self.upsert_live(item);
    }
    for record in self.records.values_mut() {
        if !live_names.contains(&record.name) && !record.hidden {
            record.mark_dead(now);
        }
    }
}
```

```rust
pub fn request_close_for_focused(&mut self) {
    if let Some(session) = self.focused_session_name() {
        if self.store.is_live(&session) {
            self.overlay = Some(OverlayState::ConfirmKill { session });
        } else {
            self.store.hide(&session);

    }
```

```rust
pub fn apply_probe_result(&mut self, result: Result<Vec<LiveSession>, AppError>) {
    match result {
        Ok(sessions) => {
            self.global_error = None;
            self.store.reconcile(sessions, Instant::now());
        }
        Err(err) => self.global_error = Some(err),
    }
}
```
